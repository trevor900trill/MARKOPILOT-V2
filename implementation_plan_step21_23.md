# Implementation Plan: Email Outreach Pipeline (Steps 21-23)

Based on Section 9 of the system architecture configuration, this plan scales out the complete outbound email connectivity and automated sending lifecycle across Steps 21 through 23.

## User Review Required

> [!IMPORTANT]
> **Global Job vs Per-Brand Job (Step 22)**: The specification designates the `OutreachWorker` to run every 30 minutes traversing all brands (`SELECT from outreach_emails`). I will enforce this uniformly as a singular global `RecurringJob` triggered at `*/30 * * * *`, which will isolate payloads within the loop per brand to ensure robust scaling limit enforcement without queuing thousands of individual Hangfire cron threads.

> [!TIP]
> **Spam Heuristics Check**: The Email generation job employs strict spam boundary checking (minimum 80 words, maximum 350 words). I will add a retry cycle inside the generation component—if OpenRouter generates an email breaking these lengths, we'll try regenerating it instantly once before gracefully aborting.

## Proposed Changes

### Step 21: Infrastructural Gmail Bindings (`OAuth & OutreachService`)
- **OAuth Expansion**: Extrapolate `Markopilot.Infrastructure/Social/OAuthService.cs` to integrate Google API `authorization_code` exchanges securely persisting the `gmail_access_token` and `refresh_tokens` under AES-256 mappings inside `Brand`.
- **IOutreachService**: Establish standard contracts in `Markopilot.Core/Interfaces/IOutreachService.cs` supporting `TokenRefresh`, `ValidateSpamScore`, and `DispatchRFC2822EmailAsync`.
- **Gmail Infrastructure Provider**: Create `Markopilot.Infrastructure/Email/OutreachService.cs` directly interfacing with Google's REST POST `/v1/users/me/messages/send` compiling the Base64 MIME boundaries required for clean RFC-2822 transmission.

### Step 22: The Outreach Worker Engine
- **Component**: Create `Markopilot.Workers/Workers/OutreachWorker.cs`.
- **Loop Architecture**: 
   - Retrieves active brands where `automation_outreach_enabled = true`.
   - Iterates through the top 10 `queued` outreach emails scheduled `<= NOW()` per brand.
   - Evaluates `SuppressionList` matches (preventing bounced contacts from tanking the sender's reputation domain).
   - Generates the Payload: Maps `Gemini 2.5 Flash` contexts merging Brand Details and Lead Attributes to build the Subject and HTML bodies explicitly overriding hallucinations.
   - Posts to Gmail and escalates statuses to 'sent'.

### Step 23: Campaigns / Outreach Frontend Dashboard
- **Component**: Create `apps/web/src/app/(dashboard)/dashboard/outreach/page.tsx`
- **Design Elements**: 
   - OAuth Connect Cards guiding users through the Gmail consent scopes (`https://www.googleapis.com/auth/gmail.send`).
   - Global Delivery limits setting configurations mirroring properties from the backend models (Daily Volumes, Delay Hours).
   - Data Table showcasing dispatched logs with granular views into specific `BodyText` copies generated by AI.

## Verification Plan
1. Ensure the `OutreachWorker` properly queues globally under the `critical` or `scale` Hangfire topologies.
2. Validate the `OutreachService` base64url generation does not encode malformed MIME types triggering Gmail API 400 responses.
